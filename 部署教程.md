# MBTI 网站部署教程

从零搭建一个公网可访问的静态网站，技术栈：GitHub + Cloudflare Pages。

---

## 一、准备工作

你需要以下账号：

| 平台 | 地址 | 作用 |
|------|------|------|
| GitHub | https://github.com | 存放代码 |
| Cloudflare | https://dash.cloudflare.com | 部署网站，提供免费域名 |

---

## 二、把代码上传到 GitHub

### 2.1 初始化本地仓库

在项目文件夹里打开终端（CMD / PowerShell / Git Bash），依次执行：

```bash
# 初始化 Git 仓库
git init

# 把所有文件加入暂存区
git add -A

# 创建第一次提交
git commit -m "Initial commit"
```

### 2.2 关联远程仓库

```bash
# 设置你的身份信息（只需设置一次）
git config user.name "你的用户名"
git config user.email "你的邮箱@qq.com"

# 把本地分支重命名为 main
git branch -M main

# 关联 GitHub 远程仓库（把下面地址换成你自己的）
git remote add origin https://github.com/你的用户名/你的仓库名.git

# 推送到 GitHub
git push -u origin main
```

以后每次修改代码后，只需：

```bash
git add -A
git commit -m "本次改了什么"
git push
```

### 2.3 备选：直接在网页上传

如果命令行推不上去（被墙），也可以：

1. 打开你的 GitHub 仓库网页
2. 点 **Add file** → **Upload files**
3. 把你的 HTML、CSS、JS 文件拖进去
4. 点 **Commit changes**

效果一样。

---

## 三、绑定 Cloudflare Pages 部署

### 3.1 创建 Pages 项目

1. 打开 https://dash.cloudflare.com 并登录
2. 左侧菜单点 **Workers 和 Pages**
3. 点右上角蓝色 **创建** 按钮
4. 选择 **Pages** 标签页
5. 点 **连接到 Git** 按钮
6. 授权 GitHub 账号，选择你的代码仓库
7. 点 **开始设置**

### 3.2 填写构建设置

| 字段 | 填写内容 |
|------|---------|
| 项目名称 | 自定义（如 `mbti-test`） |
| 生产分支 | `main` |
| 框架预设 | 选择 **无** |
| 构建命令 | 留空 |
| 构建输出目录 | 留空 |

> 因为这是纯静态网站（HTML/CSS/JS），不需要构建，所以全部留空。

8. 点 **保存并部署**

### 3.3 获取免费域名

部署成功后，Cloudflare 会自动分配一个免费域名：

```
你的项目名.pages.dev
```

这个域名全球可访问，自带 HTTPS，完全免费。

### 3.4 自动更新

之后每次你 `git push` 新代码到 GitHub 的 `main` 分支，Cloudflare 会自动重新部署，30-60 秒后网站自动更新。完全无需手动操作。

---

## 四、绑定自定义域名（可选）

如果你有自己的域名，也可以绑定到 Cloudflare Pages：

1. 进入 Pages 项目 → 顶部点 **自定义域** 标签
2. 点 **设置自定义域**，输入你的域名（如 `xxx.eu.cc`）
3. Cloudflare 会提示你在域名 DNS 里添加一条 CNAME 记录
4. 去你的域名注册商后台，添加：

| 类型 | 主机记录 | 记录值 |
|------|---------|--------|
| CNAME | `@` 或 `www` | `你的项目名.pages.dev` |

5. 回到 Cloudflare 点 **验证**，等待 DNS 生效（几分钟到几小时不等）

---

## 五、总结

```
本地写代码 → git push 到 GitHub → Cloudflare 自动部署 → 全网可访问
```

整个流程的核心就三步：

1. **git push** 把代码推到 GitHub
2. **Cloudflare Pages 连上** 这个 GitHub 仓库
3. 拿到 `xxx.pages.dev` 域名，发给别人就能打开
